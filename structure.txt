NMservices Project Structure
============================

Root:
├── .env.example          # Environment variables template
├── pyproject.toml        # Project dependencies
├── README.md             # Project documentation
├── REFACTORING.md        # Refactoring notes
│
├── src/nms/              # Main package
│   ├── main.py           # FastAPI app entry point
│   ├── config.py         # Configuration with pydantic-settings
│   │
│   ├── api/              # API Layer (Routers)
│   │   ├── dependencies.py  # Shared dependencies (auth)
│   │   ├── users.py         # User endpoints (/users/*)
│   │   └── orders.py        # Order endpoints (/orders/*)
│   │
│   ├── models/           # Data Models (Pydantic)
│   │   ├── user.py       # User request/response models
│   │   └── order.py      # Order request/response models
│   │
│   └── services/         # Business Logic Layer
│       ├── auth.py       # Authentication & registration
│       ├── order.py      # Order management
│       └── user_agree.py # (reserved)
│
└── tests/
    └── test_main.py      # API tests (all passing ✓)

Key Improvements:
- Clean Architecture with separated layers
- Centralized configuration
- Reusable services
- Modular API routers
- Backward compatibility with legacy endpoints
- All tests passing without warnings
